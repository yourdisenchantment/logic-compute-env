Многомашинные системы представляют собой класс параллельных вычислительных систем, в которых множество независимых
вычислительных узлов объединены коммуникационной сетью. Каждый узел обладает собственной локальной памятью и
взаимодействует с другими узлами посредством передачи сообщений, без прямого доступа к памяти других узлов.

**Основные характеристики**

- Распределенная память (архитектура NORMA): Каждый вычислительный узел содержит процессор, локальную память,
  операционную
  систему и сетевой интерфейс. Архитектура NORMA (No Remote Memory Access) исключает аппаратные механизмы когерентности
  кэша и требует явной коммуникации для доступа к удаленным данным.

- Передача сообщений: Основным принципом взаимодействия является передача сообщений через специализированные API: MPI
  или
  PVM. Сообщение содержит упакованные данные и маршрутизируется по сетевым каналам между узлами.

- Сетевые топологии: Узлы объединяются через статические или динамические сети с различными топологиями: кольцо, дерево,
  решетка, тор.

- Высокая масштабируемость: Архитектура позволяет строить системы из тысяч узлов, добавляя одновременно вычислительные
  ресурсы и память. Отсутствие единой шины памяти устраняет основное узкое место, характерное для систем с общей
  памятью.

- Автономность узлов: Каждый узел функционирует как независимый компьютер с собственной копией операционной системы, что
  допускает конфигурации с различными архитектурами процессоров и объемами памяти.

**Преимущества**

- Линейная масштабируемость: Добавление новых узлов приводит к пропорциональному увеличению вычислительной мощности и
  объема памяти без узких мест, характерных для SMP-систем. Суммарная пропускная способность памяти и вычислительные
  ресурсы растут линейно с числом узлов.

- Отказоустойчивость: При выходе из строя отдельных узлов остальная часть системы может продолжать работу,
  перераспределяя
  задачи на функционирующие узлы.

- Локальность памяти: Каждый процессор имеет эксклюзивный высокоскоростной доступ к своей локальной памяти без
  конкуренции
  с другими процессорами.

- Географическое распределение: Узлы могут быть физически разнесены на значительные расстояния, образуя грид-системы и
  распределенные вычислительные инфраструктуры.

**Недостатки**

- Сложность программирования: Отсутствие общего адресного пространства требует явного управления коммуникацией,
  упаковкой
  данных, синхронизацией процессов и балансировкой нагрузки.

- Высокие накладные расходы на коммуникацию: Передача данных по сети на несколько порядков медленнее, чем доступ к
  локальной памяти. Накладные расходы включают сериализацию данных, задержки в сетевом стеке ОС и конкуренцию за сетевые
  ресурсы.

- Сложность отладки и профилирования: Недетерминированность выполнения из-за вариации сетевых задержек, трудности
  воспроизведения ошибок и отсутствие глобального представления состояния системы затрудняют обнаружение состояний гонки
  и
  взаимных блокировок.

- Балансировка нагрузки: Неравномерное распределение данных или вычислений приводит к простою узлов и снижению общей
  эффективности.

- Управление и администрирование: Эксплуатация большого количества независимых узлов с собственными операционными
  системами требует сложных систем автоматизации, управления конфигурациями и мониторинга.

- Отсутствие аппаратной когерентности данных: В отличие от систем с общей памятью, реплицированных вычисленных данных
  полностью возлагается на программное обеспечение. Это требует явной реализации протоколов синхронизации и может
  приводить к избыточным коммуникациям для поддержания согласованности.
